API References

callPolling

A decorator used to poll remote endpoint with action.

Parameters

  • options object -
    • options.until Function The function to set conditions to stop the polling and return the data.
    • options.mapping Function The mapping function to transform response to the data format needed. (optional, default data=>data)
    • options.interval number Time to wait between each polling call. (optional, default 1000)
    • options.timeout number The max time to wait for the polling before abort it. (optional, default 30*1000)

Returns Function The decorated function returns the polling result.

errorCounter

A decorator to use context.metrics to count error thrown from action.

Parameters

  • options object - (optional, default {})
    • options.value function (): number Use to set counter values, e.g. In case that retries are used. (optional, default ()=>1)
    • options.extraLabels function (): object Use to add extra labels to error metrics.
    • options.parseLabel (optional, default ()=>{})

errorHandler

A decorator to add conditional side-effect before error being thrown e.g. It can be used together with error-metrics to create conditional error-metrics e.g. It can also be used to handle specific error by throw error in the handler.

Parameters

  • $0 Object (optional, default {})
    • $0.condition (optional, default ()=>false)
    • $0.handler (optional, default ()=>{})

errorMute

A decorator to mute errors when conditions are met.

Parameters

  • $0 Object (optional, default {})
    • $0.condition (optional, default ()=>true)

Returns (object | undefined) If the error is muted(not thrown to upper level) it is accessible in the return value.

errorRetry

A decorator to retry action until condition met or reach maxRetries.

Parameters

  • config (optional, default {})

Returns (object | Array) The data returned from the original action call.

errorTag

A decorator used on actions when they are not chained with a logged upper-level call this decorator attaches action name to the error then being thrown to a logged level.

Parameters

  • $0 Object (optional, default {})
    • $0.tag (optional, default (e,p,m,c,a)=>{action:a.name,message:e.message,constructor:e.constructor.name,stack:e.stack.split('\n').map(s=>s.trim())})

eventLogger

A decorator to attach standard log behaviour to action bypass: when logger instance is not available in context augment: parse action name and added to action meta to be chained in sub-actions after: log success event, with options to include param and result error: log error event, with option to parse error.

Parameters

  • $0 Object (optional, default {})
    • $0.logParam
    • $0.logResult
    • $0.errorParser (optional, default e=>e)

eventTimer

A decorator to timing action execution time in both success/error cases and send metrics using the client attached in context.

Parameters

  • $0 Object (optional, default {})
    • $0.parseLabel (optional, default ()=>{})

addHooks

A decorator creator to maintain a standard pattern so that decorator behaviour is more predictable with lgihter test.

Parameters

  • $0 Object (optional, default {})
    • $0.bypassHook (optional, default ()=>false)
    • $0.storeHook (optional, default ()=>{})
    • $0.beforeHook (optional, default ()=>{})
    • $0.actionHook (optional, default (param,meta,context,action)=>action(param,meta,context))
    • $0.afterHook (optional, default ()=>{})
    • $0.errorHook (optional, default ()=>{})

Returns Function The decorated action function that returns the result returned by the original action function or the modified one or value returned from errorHook.

stats

Used when the exhaustive values of a label is not clear e.g. When it is stats for errorCodes.

Parameters

  • labelName string The name of the label.

Returns object {[labelValue]: count}.

add

Add a new metrics unit from metrics-specs.

Parameters

  • $0 Object
    • $0.type
    • $0.name
    • $0.specs ...any

load

Load a list of metrics specs to metricsUnits.

Parameters

  • list Array List of metrics specs.

list

List the metrics unit instances stored in metricsUnits.

Returns object <{[metrics name]: metrics unit instance}>.

find

Find a metrics unit by action/type or by name(higher priority).

Parameters

  • $0 Object
    • $0.action
    • $0.type
    • $0.name

Returns object The metrics unit instance.

reset

Clear instances stored in metricsUnits and prometheus.register.

objectContains

Check if object a contains another object b by shallow equal.

Parameters

  • a object The bigger object.
  • b object The smaller object.

Returns boolean If contains.

objectOnlyKeys

Trim object keys to the target array.

Parameters

  • input object The object to be trimmed.
  • target Array The array of target keys.

Returns object The new object with trimmed keys.
